# Agentic Meal Planner

This project is an AI-powered meal planning workflow that creates a weekly dinner plan based on user constraints.

The application uses an agentic workflow to draft, critique, revise, and finalize a meal plan. After the final plan is created, it generates a grocery shopping list that excludes ingredients already available in the pantry.

## Project Goal

The goal of this project is to demonstrate how multiple Large Language Model calls can work together in a structured workflow.

Instead of asking the model for one final answer immediately, the project separates the task into smaller steps:

1. Draft a meal plan.
2. Critique the plan for rule violations.
3. Revise the plan based on fixes and suggestions.
4. Repeat the review process for a limited number of passes.
5. Generate a grocery list from the final plan.

## User Inputs

The workflow uses the following user parameters:

- Number of people
- Number of days
- Daily calorie target
- Food allergens
- Budget in USD
- Pantry ingredients already available

Example input:

```python
user_inputs = {
    "people": 2,
    "days": 7,
    "daily_calories": 2000,
    "allergens": ["peanuts", "shellfish"],
    "budget_usd": 110,
    "pantry": ["rice", "lentils", "frozen spinach"]
}
```

- Prompt chaining
- Agentic workflow design

- Evaluation and revision loops

## How It Works

### 1. Draft Plan

The `draft_plan()` function asks the model to create a weekly dinner plan using the user's requirements.

The plan must include:

- Day
- Dish name
- Ingredients
- Calories
- Estimated cost
- Total cost

### 2. Critique Plan

The `critique_plan()` function evaluates the meal plan and checks whether it follows the rules.

It checks:

- Budget limit
- Calorie target
- Allergen restrictions
- Pantry ingredient usage
- Correct number of days

The function returns JSON with two fields:

```json
{
  "fixes": [],
  "suggestions": []
}
```

### 3. Revise Plan

The `revise_plan()` function applies the mandatory fixes and optionally uses the suggestions if they do not break the rules.

### 4. Build Final Plan

The `build_plan()` function controls the full workflow. It runs the draft, critique, and revision steps until the plan is acceptable or the maximum number of passes is reached.

### 5. Build Grocery List

The `build_grocery_list()` function creates a shopping list from the final meal plan and excludes pantry items the user already has.

## Setup

Create a virtual environment:

```bash
python -m venv venv
```

Activate it:

```bash
# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate
```

Install dependencies:

```bash
pip install -r requirements.txt
```

Create a `.env` file or set your OpenAI API key in your environment:

```bash
OPENAI_API_KEY=your_openai_api_key_here
```

Run the project:

```bash
python agentic_meal_planner.py
```
